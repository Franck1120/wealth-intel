## Problemi Critici (fix subito)

### P0 - Errori Runtime / Crash

1. **`vercel.json` vuoto** - I 5 cron job (refresh-prices, refresh-macro, check-alerts, batch-score, weekly-report) non partiranno MAI su Vercel. L'intera automazione e' morta.
   - **Fix:** Aggiungere le definizioni cron in vercel.json

2. **Upsert `asset_scores` senza UNIQUE constraint** - `batch-score/route.ts:263` e `intelligence/score/route.ts:80` usano `onConflict: 'asset_id'` ma non esiste un vincolo UNIQUE su `asset_id`. Postgres dara' errore.
   - **Fix:** `ALTER TABLE asset_scores ADD UNIQUE (asset_id);` in una nuova migrazione

3. **Upsert `price_cache` con onConflict sbagliato** - `refresh-prices/route.ts:176` usa `onConflict: 'asset_symbol,date'` ma il UNIQUE e' su `(asset_symbol, asset_type, date, source)`. Il cron prezzi non funziona.
   - **Fix:** Cambiare onConflict a `'asset_symbol,asset_type,date,source'` oppure aggiungere un UNIQUE su `(asset_symbol, date)`

4. **3 route rotte** - Link a pagine inesistenti:
   - `/journal/new` (journal/page.tsx:84,111)
   - `/portfolio/[id]/transaction` (portfolio/[id]/page.tsx:152,195)
   - **Fix:** Creare le pagine o cambiare il pattern (es. modal inline)

5. **2 API endpoint mancanti:**
   - `/api/assets/[id]/price-history` - usato da `asset-price-chart.tsx:43`. Tutti i grafici prezzi sono rotti.
   - `/api/opportunities/[id]` - usato da `opportunities/page.tsx:76`. Il kanban "Move to..." non funziona.
   - **Fix:** Creare le API route

### P0 - Sicurezza

6. **`@anthropic-ai/sdk` non e' in `package.json`** ma importato in `lib/ai/client.ts:24` via `require()`. Se qualcuno abilita il modulo AI, crash garantito.
   - **Fix:** `npm install @anthropic-ai/sdk` oppure rimuovere il modulo AI

7. **Settings page: "Your key is encrypted and stored securely"** (settings/page.tsx:248) e' FALSO. Il campo `anthropic_api_key` viene strippato da Zod server-side. Feature non funzionante con copy ingannevole.
   - **Fix:** Rimuovere il campo o implementare davvero l'encryption + storage

### P0 - Legal

8. **Zero compliance GDPR:**
   - No privacy policy
   - No terms of service
   - No consenso nel login (login/page.tsx:37-66)
   - No cancellazione account
   - No export dati
   - No cookie banner
   - No link unsubscribe nelle email (lib/email/templates.ts)
   - **Fix:** Creare pagine legal, aggiungere checkbox consenso, implementare account deletion e data export

## Problemi Medi (fix presto)

### Database

9. **Missing `ON DELETE CASCADE`** su 5 tabelle: `portfolios`, `opportunities`, `decision_journal`, `alerts`, `weekly_reports` referenziano `auth.users(id)` senza CASCADE. Dati orfani se un utente viene cancellato.
   - `001_initial_schema.sql:20,116,134,149,161`

10. **Missing index su `portfolios.user_id`** - Ogni policy RLS per holdings/transactions fa una subquery su portfolios filtrata per user_id. Senza indice = full table scan.
    - `003_indexes_and_functions.sql` - aggiungere `CREATE INDEX idx_portfolios_user ON portfolios(user_id);`

### Performance

11. **Tutte le 15 pagine usano `force-dynamic`** - Zero caching. Le pagine markets/macro cambiano ogni 4h/giorno, non ad ogni richiesta.
    - **Fix:** Usare `revalidate` con intervalli appropriati (es. 3600 per markets, 86400 per macro)

12. **Analytics page: 5 query sequenziali** (analytics/page.tsx:54-91) - Tutte indipendenti, dovrebbero essere `Promise.all()`.

13. **Weekly report cron: N+1 disaster** - 6-8 query per utente nel loop. Con 1000 utenti = 8000 query in un singolo run.
    - `weekly-report/route.ts:142-310`

14. **No `loading.tsx` in tutto il progetto** - Zero indicatori durante la navigazione.

15. **No `error.tsx` / `not-found.tsx`** - Errori runtime mostrano la pagina default Next.js senza branding.

### Codice

16. **44 `any` type** in 20+ file - La causa principale e' il type file Supabase incompleto. Fix: `npx supabase gen types typescript`.

17. **Rate limiting implementato ma MAI usato** - `lib/rate-limit.ts` (94 righe) non e' importato da nessuna API route. Tutti gli endpoint sono senza protezione.

18. **15 `console.error` raw** invece di usare il logger strutturato (intelligence/score, data/prices, data/defi, data/macro).

19. **Funzioni duplicate:**
    - `chunk<T>()` in batch-score/route.ts:41 e refresh-prices/route.ts:32
    - `computeDailyReturns()` in batch-score/route.ts:52 e intelligence/score/route.ts:115
    - `delay()` in batch-score/route.ts:34 e refresh-prices/route.ts:25

20. **5 componenti mai usati:** `allocation-pie.tsx`, `price-chart.tsx`, `macro-dashboard.tsx`, `journal-timeline.tsx` (placeholder), `report-viewer.tsx` (placeholder)

### Sicurezza

21. **No Content-Security-Policy header** - Rischio XSS significativo.
    - `next.config.ts:17-28` - aggiungere CSP

22. **No `Strict-Transport-Security` (HSTS)**

23. **Middleware deprecato** - Next.js 16 avvisa: usare `proxy` invece di `middleware`.

### UX

24. **9 catch vuoti** - Errori catturati e scartati senza feedback utente:
    - opportunities/page.tsx:62,88,113
    - alerts-manager.tsx:85,103,125
    - equity-search-bar.tsx:39,61
    - transaction-form.tsx:82

25. **No conferma su azioni distruttive** - Delete alert (alerts-manager.tsx:92-105) e Sign Out (settings/page.tsx:102-108) sono istantanei.

26. **Search bar nel header non funziona** - Puramente decorativa, nessun onChange/handler (header.tsx:35-39).

27. **Mobile nav mancano 4 sezioni** - Alerts, Reports, Analytics, Settings inaccessibili su mobile (mobile-nav.tsx).

28. **Cron ignorano preferenze notifiche utente** - weekly-report e check-alerts inviano email senza controllare `user_settings.notifications_*`.

13. **Batch processing con concurrency limits** - Cron jobs ben progettati con batch + delay
14. **Italian tax calculator** unico nel panorama competitivo (FIFO, BTP 12.5%, zainetto fiscale)
15. **Decision journal con emotion tracking** - Feature unica di finanza comportamentale
